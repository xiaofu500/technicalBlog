性能方面; ​

放弃“全量加载”，拥抱“按需流式”：绝不试图一次性将所有数据加载到内存。而是像在线地图一样，只加载当前视点能看到的数据，并且随着用户的交互，动态流入流出。

统一数据范式，抽象异构源：无论数据源是倾斜摄影（OSGB）、人工模型（Revit）、点云（LAS）还是 BIM，都通过 3D Tiles 规范转换为统一的瓦片树结构。这让核心引擎可以用同一套机制处理所有类型。

渲染驱动，性能优先：所有决策（加载什么、渲染什么、丢弃什么）的最终目标都是维持高帧率。系统是一个由渲染循环控制的反馈系统，根据当前性能动态调整加载策略。

二、组件关系 ​


1. Viewer - 应用程序容器
Viewer
├── scene: Scene                    // 3D场景
├── dataSources: DataSourceCollection // 数据源集合
├── clock: Clock                    // 时间控制器
├── imageryLayers: ImageryLayerCollection // 影像图层
├── terrainProvider: TerrainProvider // 地形提供者
├── entities: EntityCollection      // 实体集合
└── widgets: Widget集合             // UI组件
    ├── animation: Animation
    ├── timeline: Timeline
    ├── baseLayerPicker: BaseLayerPicker
    └── infoBox: InfoBox
2. Scene - 3D场景
Scene
├── globe: Globe                    // 地球对象
├── camera: Camera                  // 相机
├── primitives: PrimitiveCollection // 图元集合
├── groundPrimitives: PrimitiveCollection // 地面图元
├── frameState: FrameState          // 帧状态
└── context: Context                // WebGL上下文
3.1 Entity API（实体系统）
DataSourceCollection
    ↓
DataSource (CzmlDataSource, GeoJsonDataSource等)
    ↓
EntityCollection
    ↓
Entity
├── position: PositionProperty      // 位置（支持时间插值）
├── billboard: BillboardGraphics    // 图标
├── label: LabelGraphics            // 标签
├── polygon: PolygonGraphics        // 多边形
├── polyline: PolylineGraphics      // 折线
└── model: ModelGraphics           // 3D模型
3.2 Primitive- 图元集合
PrimitiveCollection
    ↓
Primitive / GroundPrimitive
├── geometryInstance: GeometryInstance
│   ├── geometry: Geometry          // 几何定义
│   └── attributes: Object         // 顶点属性
└── appearance: Appearance
    ├── material: Material         // 材质
    └── renderState: Object        // 渲染状态
4.Entity->Primitive
Entity系统 → Primitive系统转换流程：
1. Entity属性更新 → 触发Changed事件
2. GeometryUpdater检测变化 → 创建/更新GeometryInstance
3. AppearanceUpdater更新材质属性
4. PrimitiveConverter组装为Primitive
5. 添加到Scene.primitives集合

四、模块职责划分 ​

  1. 管理层模块

    • Viewer: 应用程序入口，组件协调
    • Scene: 场景状态管理，渲染调度-
    • Clock: 时间线管理，动画驱动-
    • DataSourceCollection: 多源数据聚合

  2. 数据模型模块

    • Entity系统: 面向对象的地理要素表示
    • Property系统: 时间动态属性管理
    • Geometry系统: 几何形状定义
    • Material系统: 材质外观定义

  3. 渲染引擎模块

    • Primitive系统: WebGL渲染抽象
    • Shader系统: 着色器管理
    • Batch系统: 渲染批处理优化
    • State系统: WebGL状态管理

  4. 地球引擎模块

    • Globe: 地球对象核心
    • TerrainProvider: 地形数据接口
    • ImageryProvider: 影像数据接口
    • Tile系统: 瓦片加载调度

  5. 基础设施模块

    • 数学库: Cartesian3, Matrix4, Quaternion等
    • 坐标转换: 经纬度←→笛卡尔坐标
    • 网络请求: RequestScheduler, ResourceLoader
    • 事件系统: 观察者模式实现 五、关键设计模式
