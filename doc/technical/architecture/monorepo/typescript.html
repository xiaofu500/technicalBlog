typescript 类型编辑脚本语言，

相对 js 优势 ​

编码层面 ​

  • 静态类型检查
  • 代码提示
  • 代码跳转
  • 类似 C++的类型系统,需要先定义类型,再使用.使用 rollup 等工具可以将 typescript 代码编译为.d.ts 文件,用于类型检查,后使用 babel,vitepress 可以生成 API 文档

l ​

es6 语法： enum class interface

装饰器（Decorator） ​

  • 需要在 tsconfig.json 中开启装饰器功能 使用装饰器可以在不改变原有代码的情况下,添加额外的功能.
  • 装饰器可以用于类、方法、属性、参数等.
  • 经验：
    • 可用于泛型约束
    • 可用于依赖注入
    • 可用于 AOP 编程
    • 可用于日志记录（nestjs 较为常用）
    • 可用于性能监控(集成性能监控平台,如 NewRelic, Datadog 等，...后期开发性能监控这块留个demo)
typescript
function log(target: any, key: string, descriptor: PropertyDescriptor) {
  const originalMethod = descriptor.value;
  descriptor.value = function (...args: any[]) {
    console.log(`Calling ${key} with arguments:`, ...args);
    return originalMethod.apply(this, args);
  };
  return descriptor;
}

class MyClass {
  @log()
  myMethod(arg1: string, arg2: number) {
    console.log("myMethod is called with arguments:", arg1, arg2);
  }
}
