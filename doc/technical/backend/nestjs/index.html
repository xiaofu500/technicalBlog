service ​

服务是 NestJS 中最核心的概念之一，它负责处 理应用程序中的业务逻辑。服务可以定义为类，并使用 @Injectable() 装饰器进行标记。服务可以注入其他服务或模块，并使用 @Inject() 装饰器进行注入。

typescript

//服务
@Injectable()
export class HelloService {
  getHello(): string {
    return 'Hello World!';
  }
}
//服务注入
@Module({
  imports: [],
  controllers: [HelloController],
  providers: [HelloService],
})
export class HelloModule { }

middleware ​

中间件是 NestJS 中的一种特殊类型的服务，它可以在请求处理管道中拦截和处理请求。中间件可以用于执行一些通用的任务，比如日志记录、身份验证、授权等。

typescript
//创建中间件
@Injectable()
export class LoggerMiddleware implements NestMiddleware {
      use(req: Request, res: Response, next: NextFunction) {
            Logger.log(`Request...[INFO]${req.url},${req.method},${req}`);
            next();
            Logger.log(`Response...[INFO]${res.status},${res}`);
      }
}

中间件使用 ​

  1. 全局中间件：全局中间件会在所有路由处理程序之前执行。
  2. 路由中间件: 路由中间件会在特定路由处理程序之前执行。
  3. 模块中间件：模块中间件会在模块中的所有路由处理程序之前执行。
  4. 函数中间件 ：函数中间件是在 NestJS 中最常用的一种中间件类型。
  5. 异步中间件: 异步中间件允许您使用 async/await 语法来处理中间件。
  6. 错误处理中间件 ：错误处理中间件会在应用程序抛出异常时执行。
  7. 静态资源中间件: 静态资源中间件允许您从文件中提供静态资源。
  8. 验证中间件: 验证中间件用于验证请求中的数据是否有效。
typescript
@Module({
  imports: [UserModule, AuthModule],//模块注入
  controllers: [AppController, UserController],//控制器注入
  providers: [AppService, UserService],//服务注入 
})
export class AppModule {
  configure(consumer: MiddlewareConsumer) {
    consumer.apply(LoggerMiddleware).forRoutes({ path: 'auth/login', method: RequestMethod.POST });
  }
}

洋葱模型 ​

洋葱模型是一种软件设计模式，它允许您通过将多个处理程序链接在一起，来处理请求。每个处理程序都可以在请求进入和离开时执行一些操作。

filtes ​

过滤器是 NestJS 中一种特殊类型的服务，它可以在处理请求和返回响应时执行一些操作。过滤器可以处理错误，并返回自定义的错误信息。 过滤器可以定义为类，并使用 @Injectable() 装饰器进行标记。过滤器可以注入其他服务或模块，并使用 @Inject() 装饰器进行注入。

关键点 ​

  1. 创建过滤器：创建过滤器的方法与创建服务一样，使用 @Injectable() 创建一个类，并使用 @Catch() 创建一个过滤器。
typescript
//定义
@Catch(HttpException)
export class HttpErrorFilter implements ExceptionFilter {
      catch(exception: HttpException, host: ArgumentsHost) {
            //在host中获取上下文
            const ctx = host.switchToHttp();
            //获取响应体和请求体
            const response = ctx.getResponse();
            const request = ctx.getRequest();
            //获取状态码
            const status = exception.getStatus();

            response.status(status).json({
                  statusCode: status,
                  timestamp: new Date().toISOString(),
                  path: request.url,
                  detail: exception.getResponse(),
            })
      }

}

pipes ​

管道是 NestJS 中一种特殊类型的服务，它可以在处理请求和返回响应时执行一些操作。管道可以验证请求中的数据，并将其转换为应用程序可以使用的格式。 创建管道：创建管道的方法与创建服务一样，使用 @Injectable() 创建一个类，并使用 @Pipe() 创建一个管道。

typescript
//定义
//约定的事情放在管道中

//用来验证或者转换数据


import { ArgumentMetadata, Injectable, PipeTransform } from "@nestjs/common";

@Injectable()
export class DetailPipe implements PipeTransform {





      //可以验证和转换
      /**
       * 
       * @param value 管道接收的值
       * @param metadata 元数据
       * @returns 
       */
      //metadata元数据
      transform(value: any, metadata: ArgumentMetadata) {
            //1.验证
            if (typeof value !== "string") {
                  throw new Error("value is not string");
            }
            //2.转换
            value = JSON.parse(value);
            return value;
      }
}

//使用
@Controller('auth')
export class AuthController {
  constructor(private readonly authService: AuthService) { }
  @Post('login')
  //此处使用管道来验证参数
  @UsePipes(new DetailPipe())
  login(@Body() body: AuthParams): string {
    //s数据库是否存在
    console.log(body, "=====body");
    return this.authService.login();
  }
}

pg数据库 ​

  1. 安装pg数据库,通过docker安装
bash
1. 创建一个docker-compose.yml文件，用于配置pg数据库容器。
      docker pull docker.io/library/postgres:18.1
2. 运行docker镜像
    
docker run --name miaoma-nest-pg \
  -p 5432:5432 \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_DB=miaoma_db \
  -v miaoma-postgres-data:/var/lib/postgresql/data \
  -d \
  postgres:18.1

  3. 创建一个data目录，用于保存数据库数据。
  mkdir data
  chmod 777 data
参数含义:
其中-e => environment variables 
--name miaoma-nest-pg 容器名称
-p 5432:5432 端口映射
-e POSTGRES_USER=postgres 数据库用户名
-e POSTGRES_DB=miaoma_db 数据库名
-e ALLOW_EMPTY_PASSWORD=yes 允许空密码
-v miaoma-postgres-data:/var/lib/postgresql/data 卷挂载
-d 后台运行
postgres:18.1 镜像名
  1. 配置pg数据库：
    1. 创建一个docker-compose.yml文件，用于配置pg数据库容器。
    2. 在docker-compose.yml文件中，定义pg数据库容器的配置，包括端口映射、环境变量、卷挂载等。
    3. 运行docker-compose up -d命令，启动pg数据库容器。
  2. 安装pg数据库模块，并连接数据库：
    1. 安装@nestjs/typeorm和typeorm模块。
    2. 创建一个typeorm.config.ts文件，用于配置数据库连接。
    3. 在typeorm.config.ts文件中，定义数据库连接的配置，包括数据库类型、主机、端口、用户名、密码、数据库名等。
    4. 在app.module.ts文件中，导入TypeOrmModule.forRoot()方法，并传递typeorm.config.ts文件的路径。
