NestJS 学习笔记 ​

核心概念(6) ​

  • module
    • 控制器 controller
    • 服务 service(依赖注入)
  • middleware(中间件)
  • pipe(管道)
  • guard(守卫)
  • interceptor(拦截器)

模块 module ​

模块是 NestJS 应用程序的基本构建块，用于组织和管理应用程序的不同部分。每个模块都有自己的控制器、服务、依赖注入等。

typescript
import { Module } from '@nestjs/common';

@Module({
  imports: [],
  controllers: [],
  providers: [],
  exports: [],
})
export class AppModule {}

控制器 controller ​

控制器负责处理 HTTP 请求和响应。每个控制器都有一个或多个路由处理方法，用于处理不同的 HTTP 请求。

typescript
import { Controller } from '@nestjs/common';

@Controller()
export class AppController {
  constructor(private readonly appService: AppService) {}
}

服务 service ​

服务是用于执行业务逻辑的类。它们通常负责处理数据访问、业务规则等。服务可以在控制器中注入，用于处理业务逻辑。

typescript
import { Controller } from '@nestjs/common';

@Injectable()
export class AppService {
  constructor(private readonly appService: AppService) {}
}

middleware(中间件) ​

中间件是在请求和响应周期中执行的函数。它们可以用于修改请求和响应对象，执行额外的逻辑等。

typescript
//定义
import { Injectable, NestMiddleware } from '@nestjs/common';

@Injectable()
export class LoggerMiddleware implements NestMiddleware {
  use(req: any, res: any, next: () => void) {
    console.log('Request...');
    next();
  }
}
//使用
//在模块中使用
import { Module } from '@nestjs/common';
import { LoggerMiddleware } from './logger.middleware';

@Module({
  controllers: [],
  providers: [],
  exports: [],
})
export class AppModule {
  configure(consumer: MiddlewareConsumer) {
    consumer.apply(LoggerMiddleware).forRoutes({
      path: '/app',
      method: RequestMethod.GET,
    });
  }
}

pipe(管道) ​

管道是用于转换和验证请求数据的类。它们可以在控制器方法中使用，用于处理输入数据的转换和验证。

typescript
//定义
import { Injectable, PipeTransform } from '@nestjs/common';

@Injectable()
export class ValidationPipe implements PipeTransform {
  transform(value: any) {
    // 转换和验证逻辑
    return value;
  }
}
//使用
//在控制器方法中使用
import { Controller, Post, Body, UsePipes } from '@nestjs/common';
import { ValidationPipe } from './validation.pipe';

@Controller('app')
export class AppController {
  @Post()
  @UsePipes(new ValidationPipe())
  create(@Body() createAppDto: CreateAppDto) {
    // 处理创建应用程序的逻辑
  }
}

guard(守卫) ​

守卫是用于保护路由处理方法的类。它们可以在控制器方法执行之前进行身份验证、授权等操作。

typescript
//定义
import { Injectable, CanActivate, ExecutionContext } from '@nestjs/common';

@Injectable()
export class AuthGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean {
    // 身份验证和授权逻辑
    return true;
  }
}
//使用
//在控制器方法中使用
import { Controller, Post, Body, UseGuards } from '@nestjs/common';
import { AuthGuard } from './auth.guard';

@Controller('app')
export class AppController {
  @Post()
  @UseGuards(AuthGuard)
  create(@Body() createAppDto: CreateAppDto) {
    // 处理创建应用程序的逻辑
  }
}

interceptor(拦截器) ​

拦截器是用于在请求和响应周期中拦截和处理请求的类。它们可以用于修改请求和响应对象，执行额外的逻辑等。

typescript
//定义
import { Injectable, NestInterceptor, ExecutionContext } from '@nestjs/common';
//使用
//在模块中使用
import { Module } from '@nestjs/common';
import { LoggerInterceptor } from './logger.interceptor';

@Module({
  controllers: [],
  providers: [],
  exports: [],
})
export class AppModule {
  configure(consumer: MiddlewareConsumer) {
    consumer.apply(LoggerInterceptor).forRoutes({
      path: '/app',
      method: RequestMethod.GET,
    });
  }
}

安装 NestJS 依赖 ​

bash
npm install --save @nestjs/common @nestjs/core @nestjs/platform-express

核心模块 ​

  1. @nestjs/common：提供 NestJS 应用程序的基本功能，如控制器、服务、模块等。
  2. @nestjs/core：NestJS 应用程序的核心模块，负责应用程序的启动和生命周期管理。
  3. @nestjs/platform-express：NestJS 应用程序的 Express 平台模块，用于处理 HTTP 请求和响应。

学习重点在于思路脉络的学习

1.用户登录中的流程
    1.1 用户输入用户名和密码
    1.2 后端验证用户名和密码是否正确
    1.3 如果验证通过，后端生成一个 token 并返回给前端
    1.4 前端将 token 存储在本地（如 localStorage 或 cookie）
    1.5 后续的请求中，前端将 token 放在请求头中发送给后端
    1.6 后端验证 token 的有效性
    1.7 如果 token 有效，后端返回请求的数据
    1.8 如果 token 无效，后端返回错误信息
    1.9 前端收到错误信息后，提示用户重新登录
    1.10 前端重新登录后，重复以上流程
 2. 注册流程
    2.1 用户输入用户名、密码和确认密码
    2.2 前端验证用户名、密码和确认密码是否符合要求
    2.3 如果验证通过，前端将用户名、密码和确认密码发送给后端
    2.4 后端验证用户名是否已存在
    2.5 如果用户名已存在，后端返回错误信息
    2.6 如果用户名不存在，后端创建新用户并返回成功信息
    2.7 前端收到成功信息后，提示用户登录
    2.8 前端用户登录后，重复以上流程
3. 权限验证流程
    3.1 前端发送请求到后端
    3.2 后端验证请求头中是否包含有效的 token
    3.3 如果 token 有效，后端根据用户角色和权限判断是否允许访问
    3.4 如果 token 无效或用户没有权限，后端返回错误信息
    3.5 前端收到错误信息后，提示用户重新登录或联系管理员
4. 角色管理流程
    4.1 管理员登录系统
    4.2 管理员进入角色管理页面
    4.3 管理员可以创建新角色、编辑角色权限、删除角色等
    4.4 管理员将角色分配给用户
    4.5 用户登录系统后，根据分配的角色进行权限验证
